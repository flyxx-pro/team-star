# 星图测控成功经验总结

> 从1家公司跑通到8家公司可复用的方法

---

## 一、星图测控为什么做成了

### 1. 先读PDF再写代码
- 翻开第47页，找到"发行前后公司的股本情况"这张表
- 对着PDF一行一行抄进Excel，持股数量精确到个位
- 手工验证：38250000+23450781+9174219+4125000+3000000+2790000+1710000=82500000 ✓
- **这是gold standard的基础——不是AI抽的，是人工核对PDF原文的**

### 2. 五步cross-check全通过

| 步骤 | 验证 | 结果 |
|------|------|------|
| 定向发行总量 | 7500+750=8250万 | ✓ |
| IPO总量 | 8250+2750=11000万 | ✓ |
| 老股东不变 | 7家发行前后持股数完全相同 | ✓ |
| 新股东=认购 | 279+171+300=750万 | ✓ |
| 比例合计 | 发行前100%，发行后100% | ✓ |

**步骤3是星图测控特有的发现**：IPO前后7个老股东的持股数完全相同——他们一没有减持，只是被稀释。这个发现来自手工加总，不是AI。

### 3. 四个老师指定的方向全部讲透
- 定向发行：段落式披露（第35页），flow在35页、stock在47页，两页对照
- 员工持股平台：四个平台占38.88%，分批次设立，必须区分investor_type
- 发行前股东结构：7个股东精确到个位，同一张表左右对照，发行前后持股不变
- 流量存量cross-check：五步验证，每步都能回到PDF原文页码

## 二、可复用的方法

### 方法1：TOC目录定位
- 先解析目录页，找到"发行人基本情况"起始页
- 定位到第35-47页（定向发行+股权结构）
- **比全文输入省90% token，准确度反而更高**

### 方法2：持股比例合计校验
- 所有时点的持股比例加起来，不等于100%就是有缺口
- 不需要任何外部数据，小学数学
- 星图测控100%，但这不是常态——友升/云汉都有缺口
- **这是最有效的质量检查，发现过44.55%的数据缺口**

### 方法3：段落式vs表格式分Prompt
- 第35页（段落）：Prompt告诉AI"不要均分，原文没给个体数据就填null"
- 第47页（表格）：Prompt告诉AI"从展平文本重建行列关系"
- **两种策略用反的后果：对段落用表格策略→胡乱均分；对表格用段落策略→漏掉数值**

### 方法4：人机协作的边界
- AI能做的：提取框架（日期、事件类型、股东名称）、计算交叉验证
- AI不能做的：从MinerU丢失的表格中找回数值（第47页表格MinerU完全没解析出来）
- **人工必须做的：翻开PDF逐页核对表格数字，抄进Excel**
- **不能做的：凭空编造数字**

## 四、关键教训

1. **打开PDF看原文，比调Prompt有用一万倍。** 中科星图51%不是100%、44.55%的数据缺口——这两件事都是人工加总发现的，不是AI。

2. **持股比例加起来必须等于100%。** 这是最简单也最有效的校验。星图测控做到了，但代价是用户手动从PDF抄了完整表格。

3. **MinerU对表格的处理是硬伤。** 第47页的表格数据MinerU完全没解析出来，只有文字段落。表格数据的行列重建目前只能靠人工。

4. **同一种方法在不同公司上效果不同。** 星图测控段落式披露AI抽得很好，但友升的表格式披露AI也抽得很好。关键不是选一个方法，而是先判断特征再选方法。
