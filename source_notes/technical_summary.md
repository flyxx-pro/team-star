# 招股书股本结构提取 — 技术路径总结

## 完整流水线

```
① PDF下载(cninfo爬虫) → ② MinerU解析 → ③ 目录定位(TOC) → ④ 章节精准提取
→ ⑤ 关键词密度评分 → ⑥ Deepseek AI抽取 → ⑦ Pydantic校验 → ⑧ Cross-check → ⑨ Excel输出
```

## 各步骤要点

### ① PDF下载 — cninfo_spider.py
- API: `POST http://www.cninfo.com.cn/new/hisAnnouncement/query`
- **关键参数**: `searchkey="{公司名} 招股说明书"`，不加 `column` 参数
- 反爬: Session Cookie + UA轮换 + 随机延时(2-5s) + 3次重试 + ≥100KB校验
- 版本筛选: 正式稿 > 注册稿 > 申报稿 > 上会稿

### ② MinerU解析
- **页码范围是第一决定因素**。填错范围，后面全白做
- 策略：先试 `1-200`（前200页通常包含所有核心章节）
- 如果PDF超过400页，拆成 `1-200` + `200-400` 两份
- **扫描版PDF需要OCR模式**，否则解析结果为空

### ③ 目录定位（TOC）— 新增核心步骤
- **先解析目录页**（通常在前15页），提取章节→页码映射
- 找到"第四节/第五节 发行人基本情况"的起始页
- 找到下一节的起始页作为结束边界
- 只提取这个范围内的页面（通常30-80页）

### ④ 章节精准提取
- 关键词：`发行人基本情况` `历史沿革` `股本形成` `历次增资` `股权转让`
- **不要全文输入AI**——只输入目标章节的页面
- 提取前验证关键词覆盖：`增资` `股权转让` `持股比例` ≥ 3次

### ⑤ 关键词密度评分
- 对每页打分：命中关键词数 = 得分
- 前5页强制保留（封面、日期、概要）
- 按分数降序选取，控制总输入在 40K-55K 字符
- 比全文输入节省 10 倍 token，信息密度提升 10 倍

### ⑥ Deepseek AI抽取
- 模型: deepseek-chat, Temperature=0.1, max_tokens=16384
- **数据量大时分轮调用**: 认缴流量和股权存量分开提取
- Prompt要点: 不编造 + 原文证据 + 单位写列名 + 每个认购方一行
- null值根因: MinerU将PDF表格展平为纯文本后丢失行列对应关系

### ⑦-⑧ 校验
- Pydantic: 字段类型、必填项、持股比例0-100范围
- Cross-check: 上一时点股本 + 本次新增 = 下一时点股本
- evidence_text 必须能在原文中找到

### ⑨ Excel输出
- Sheet1: 认缴流量 | Sheet2: 股权结构存量 | Sheet3: Cross-check数字
- 列名固定，数字不带单位，null留空

## 常见问题与解法

| 问题 | 根因 | 解法 |
|------|------|------|
| MinerU结果全是空页 | 页码范围错误 | 先看PDF总页数，TOC定位后再设范围 |
| AI返回0条 | 文本截断或Prompt不适配 | 分轮调用，先认缴后存量 |
| 认购数量/金额全null | MinerU表格展平 | 人工查PDF补数据，或换OCR工具 |
| 时点标签不一致 | 业务理解差异 | 统一用t0/t1/t2，t0=报告期初 |
| 股权转让混入增资 | Prompt未区分 | 增资=发行人收钱发新股，转让=股东之间交易 |

## 扩展到新公司的检查清单

1. [ ] PDF总页数 < 400？→ 单次MinerU
2. [ ] 先解析TOC，确认"发行人基本情况"起始页
3. [ ] 提取目标页后验证关键词: `增资` ≥ 3次?
4. [ ] AI提取后检查: t0存在? null值 < 20%?
5. [ ] Cross-check: 前后股本变化能对上?
