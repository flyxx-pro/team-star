# 50家扩展检查清单

> 基于8家实战总结。每家公司必须通过全部检查才能标记为"完成"。

---

## 第一步：PDF获取（规则）

- [ ] PDF已下载，文件≥100KB
- [ ] 从交易所/巨潮资讯获取，非第三方网站
- [ ] 文件名含股票代码和公司简称

## 第二步：MinerU Markdown导出（规则）

- [ ] Markdown文件≥50,000字符
- [ ] 首页不是"财务会计报告"或"备查文件"（说明页码范围对）
- [ ] "发行人基本情况"关键词≥2次
- [ ] "增资"或"股权转让"关键词≥3次
- [ ] 至少5张HTML `<table>`标签

## 第三步：股权表提取（规则）

- [ ] 找到"报告期初"或"设立时"的100%股权表 → t0
- [ ] 找到每次增资后的100%股权表 → t1, t2, t3...
- [ ] 找到"发行前"的100%股权表 → 发行前
- [ ] **每张表比例合计必须在95%-105%之间，否则向用户报告**
- [ ] 每张表的时点标签使用 Markdown 前文章节标题，不要用 t0/t1/t2

## 第四步：认缴流量提取（规则+AI）

- [ ] Markdown 表格中有认购明细的，优先用正则提取（0 token）
- [ ] 段落文字中的认购信息用 AI 提取
- [ ] 检查：每个增资轮次都有对应的认购方记录？
- [ ] 检查：认购量合计 = 存量变化量？（允许小幅偏差）
- [ ] HTML 表格有 rowspan/colspan 的，先展开再解析

## 第五步：数据质量（规则）

### 必须通过的校验（不过就报告）
- [ ] **每张股权表的持股比例合计在95%-105%之间**
- [ ] 发行前后总股本变化 = 发行量（偏差<1%）
- [ ] 新股东认购量 = 发行后持股量（偏差<1%）
- [ ] 老股东发行前后持股数一致（未参与认购的）

### 需要人工确认的标记
- [ ] 股权表比例合计＜95% → 报告：可能有股东被遗漏
- [ ] 股权表比例合计＞105% → 报告：可能有重复记录
- [ ] 认购表中间有空值 → 报告：PDF 未披露，确认不是 rowspan 导致
- [ ] LLC 阶段"认购数量"为 null → 确认正确

## 第六步：investor_type标注（人工/AI）

- [ ] 每个认购方标注 investor_type（PE/VC/员工持股平台/产业资本/自然人/政府基金）
- [ ] PE 基金如有备案编码，提取到 pe_fund_detail
- [ ] 员工持股平台标注入股批次（本次新增/此前已存在）

## 第七步：输出（规则）

- [ ] JSONL 含四类记录：subscription_flow + equity_snapshot + share_transfer_flow + pe_fund_detail
- [ ] Excel 含三表：认缴流量 + 股权存量 + schema_cross_check
- [ ] 每条记录有 PDF 页码或 Markdown 表格来源标注
- [ ] 每条记录有 source 和 review_status
