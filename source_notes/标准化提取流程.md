# 标准化提取流程（三周实战总结）

## 完整流水线

```
① PDF下载(cninfo爬虫)
② MinerU JSON导出 → TOC目录定位 → 章节精准提取 → 密度评分 → Deepseek AI抽取
③ MinerU Markdown导出 → 正则解析HTML表格 → Python写入JSONL
④ 段落(AI) + 表格(正则) 合并 → Pydantic校验 → Cross-check → Excel输出
```

## 分步详解

### 第一步：MinerU双模导出
- **JSON模式**：保留页码标记，用于段落文字提取和证据溯源
- **Markdown模式**：保留HTML `<table>`，用于表格数据提取
- 每家公司两种格式各导出一份

### 第二步：TOC目录定位（JSON模式）
- 扫描前30页找到目录页
- 正则提取"第X节 XXXX → 第N页"映射
- 定位"发行人基本情况"起止页码范围
- 验证关键词覆盖（增资/股权转让/持股比例 ≥ 3次）

### 第三步：AI段落提取（JSON模式）
- 密度评分压缩文本（7.5万→4.8万字符）
- Deepseek抽取认缴流量 + 股权存量 + 股权转让
- Temperature=0.1，数据量大时分轮调用
- 输出JSONL，字段含PDF页码和原文证据

### 第四步：正则表格提取（Markdown模式，0 token）
- 搜索关键表头文本（如"发行前后公司的股本情况如下"）
- 正则提取紧随其后的 `<table>...</table>`
- Python解析HTML → 写入JSONL
- 100%准确，零成本

### 第五步：合并校验
- 段落AI结果 + 正则表格结果 → 去重合并
- Pydantic Schema校验
- Cross-check：持股比例合计=100%、总股本前后一致性、逐股东验证
- 比例不等于100% → 自动标记 → 进入人工复核

## 分工原则

| 做什么 | 用什么 | 谁做 |
|--------|--------|------|
| PDF下载 | cninfo爬虫 | 规则 |
| 章节定位 | TOC目录法 | 规则 |
| 文本压缩 | 密度评分 | 规则 |
| 段落文字→结构化 | Deepseek AI | LLM |
| 表格数据→结构化 | Python正则解析HTML | 规则 |
| 字段校验 | Pydantic | 规则 |
| 交叉验证 | Cross-check | 规则 |
| 表格遗漏补充 | Markdown导出→正则 | 规则 |
| 无法自动提取的数据 | 人工翻PDF抄Excel/Markdown | 人工 |

## 表格遗漏检查清单

每次提取后逐项验证：
- [ ] 持股比例合计 = 100%？（不等→有遗漏）
- [ ] 总股本 = 各股东持股数之和？（不等→有遗漏）
- [ ] 发行前后各股东持股数一致？（不一致→提取错误）
- [ ] 认缴认购量合计 = 存量变化量？（不等→流量或存量有误）
- [ ] 表格block的lines是否为空？（为空→JSON模式表格丢失，需Markdown补）
