## 完整流水线（V4 最终版）

```
① PDF下载(cninfo爬虫)
② MinerU Markdown导出（唯一格式，替代JSON+Markdown双模）
③ TOC目录定位 → 正则提取章节页码映射 → 精准截取目标章节
④ 表格数据：正则解析HTML <table> → Python直接写入JSONL（0 token, 100%准确）
⑤ 段落文字：关键词密度评分压缩 → Deepseek AI抽取（Temperature=0.0）
⑥ 校验Agent（子Agent检查）：逐条验证"是否存在推算值/均分/编造" → 违规自动标记
⑦ Pydantic Schema + Cross-check（持股比例合计=100%、总股本一致性）
⑧ Excel三表输出（认缴流量 + 股权存量 + schema_cross_check）
```

## 分步详解

### 第一步：MinerU Markdown 统一导出
- **仅使用 Markdown 模式**（老师要求：以后文字和表格全部用 Markdown 解析）
- Markdown 保留完整段落文字 + HTML `<table>` 标签
- 页码标记问题：Markdown 无自动页码标记，通过 TOC 定位后的起止位置反推页码范围
- 每家公司导出 1 份即可，取消 JSON 模式

### 第二步：TOC 目录定位
- 扫描 Markdown 前 30 页找到目录区域
- 正则提取"第X节 XXXX → 第N页"映射
- 定位"发行人基本情况"起止页码范围
- 验证关键词覆盖（增资/股权转让/持股比例 ≥ 3 次）

### 第三步：表格正则提取（0 token）
- 搜索关键表头文本（如"发行前后公司的股本情况如下"）
- 正则提取紧随其后的 `<table>...</table>`
- Python 解析 HTML → 写入 JSONL
- 100% 准确，零成本

### 第四步：AI 段落提取（节省 88% Token）
- 密度评分压缩文本（去表格后约 6000 token）
- Deepseek 抽取认缴流量 + 股权存量 + 股权转让
- Temperature=0.0（最大化确定性，配合"未披露就填 null"硬约束）
- 数据量大时分轮调用

### 第五步：校验 Agent 检查（新增，老师建议）
- **参考自动写论文 workflow 中的检查监督机制**，设计独立的校验子 Agent
- 校验 Agent 的职责：逐条检查 AI 抽取结果中是否存在以下违规——
  - 原文只披露总量但 AI 均分了个体数值（如三协电机 530 万→各 265 万）
  - 原文未披露但 AI 自行推算或编造了数值
  - 有限责任公司阶段错误填写了"认购数量（万股）"
- 违规发现后：自动回退 → 标记为待复核 → 写入 manual_review_queue → 通知人工介入
- 校验 Agent 使用独立 Prompt，Temperature=0.0，只做判断不做生成

### 第六步：合并校验
- 段落 AI 结果 + 正则表格结果 + 校验 Agent 标记 → 合并
- Pydantic Schema 校验
- Cross-check：持股比例合计=100%、总股本前后一致性、逐股东验证
- 比例不等于 100% → 自动标记 → 进入人工复核

## "PDF 未披露就留空"的双重保障机制（老师 7.3 反馈）

### 第一重：Prompt 层硬约束
```
Temperature = 0.0（最大化确定性）
System Prompt 中明确：
  - "原文只披露总量而未披露个体数量时，各认购方的数量标记为 null，不要自行均分"
  - "原文未提及的任何数值均填 null，不要推算、不要编造"
  - "有限责任公司阶段，认购数量（万股）始终填 null"
```

### 第二重：校验 Agent（子 Agent 检查机制）
参考自动写论文 workflow 中的监督校验设计：

```
主 Agent（抽取）→ 输出 JSONL
       ↓
校验 Agent（检查）→ 逐条扫描：
  1. 原文包含"合计""共""总额"等总量词，但 JSONL 中个体数值相加 = 总量？
     → 如果是 → 违规：AI自行均分 ✓ 标记为待复核
  2. JSONL 中的数值在原文中找不到对应原文？
     → 如果找不到 → 违规：AI编造 ✓ 标记为待复核
  3. 有限责任公司阶段的记录中"认购数量(万股)"不为 null？
     → 如果是 → 违规：公司类型误判 ✓ 标记为待复核
       ↓
发现违规 → 自动回退 → 写入 manual_review_queue → 通知人工
无违规 → 通过 → 进入下一步
```

### 与自动写论文 Workflow 的类比
自动写论文的监督机制通常是：
- 生成 Agent 写初稿 → 审核 Agent 检查逻辑/引用/数据 → 不通过则回退重写

我们的校验 Agent 逻辑相同：
- 抽取 Agent 生成 JSONL → 校验 Agent 检查"未披露→留空"合规性 → 不通过则标记待复核

## 表格遗漏检查清单

每次提取后逐项验证：
- [ ] 持股比例合计 = 100%？（不等→有遗漏）
- [ ] 总股本 = 各股东持股数之和？（不等→有遗漏）
- [ ] 发行前后各股东持股数一致？（不一致→提取错误）
- [ ] 认缴认购量合计 = 存量变化量？（不等→流量或存量有误）
- [ ] 校验 Agent 是否标记了违规记录？（有→人工介入）
- [ ] 有限责任公司阶段认购数量是否正确为 null？
