# 第五周修改意见与第六周提交要求

## 一、本周任务

### （1）继续处理规定的 8 家公司，每位同学独立完成

如果上周处理过程中遇到困难或瓶颈，本周可以参考其他同学的处理方式和结果，相互学习，但不能直接复制他人的数据或代码。参考之后，仍需独立回到 PDF 核实，并在自己的 GitHub 仓库中完成代码和结果。

### （2）分组进行检查

| 小组 | 成员 |
|---|---|
| 第一组 | 赵秉清、杨苗鑫 |
| 第二组 | 尚恬、李泽润 |
| 第三组 | 刘宇轩、陈雨昂、霍泓锟 |

我们分成小组，然后互相检查。因为我们的目标都是 8 家企业，所以说如果处理的内容是一致的话，那可以得到一致的结果。这个任务的目的，是小组里每个人检查对方的处理，或者两个同学、小组里的同学对一下处理结果，然后分析处理结果的差异，最终达成一致。

这个过程中，大家可能会遇到处理结果的偏差。也就是尽管都是 8 家公司，但在处理过程中情况可能会有差异。这个时候，小组的同学要能够去做一下分析，看一看是哪里出现了问题，这样能够更有效地推进处理结果。

### （3）撰写一份目标公司的融资报告

2025届研究生同学做的CVC报告，大家要在你们处理的这家8家公司的基础之上，写一个简单的上市前的融资报告。

1. 什么样的公司经过多少轮投资？
2. 每一轮的投资者是什么样的情况？
3. 有什么样的共同特征？有什么不一样的地方？
4. 有什么可以总结的模式或者是方式？
   
那是让大家提取完数据之后，就不能够说不看这些数据，而是要对这些数据进行详尽的分析。

当然目前来讲我们是做的 8 家公司的测试，如果说拓展到市场里面所有的企业，那么可能得出更为详尽的分析，就像我给大家发的上届同学做的 CVC 的报告一样，是为这样的报告去奠定基础，同时能够去深入到业务内部去了解融资的结构和融资的方式。

## 三、第六周 GitHub 提交要求

每位同学提交自己的 GitHub 仓库，不提交小组公共仓库。本周至少包含：

```text
README.md                      # 运行方法、完成状态和已知问题
requirements.txt              # 主要依赖版本
data/                          # PDF/Markdown清单及可复跑输入
pipeline/                      # 完整自动提取代码和统一入口
prompts/                       # Prompt、模型参数和调用记录
manual_gold/                   # 人工Gold，不要求代码生成
auto_output/                   # 未经人工修改的raw和Auto三表
final/                         # 人工及组内复核后的三表和Excel
validation/                    # schema、Cross-check、Auto-vs-Gold和复核队列
review/                        # 组内差异、处理结论和PR/Issue链接
report/week6_report.md         # 上市前融资分析报告
logs/                          # 运行记录、输出数量和文件哈希
```

代码可复现要求：

1. README 提供一条统一运行命令，说明输入、输出和人工介入位置。
2. 使用相对路径或命令行参数，不得写死本人电脑目录。
3. 自动流程必须从 PDF 或页码化文本生成 Auto 三表、schema 结果和 Cross-check，不得从 Gold 或 Final 开始。
4. 使用 MinerU 或 LLM 时，提交调用代码、Prompt、模型参数和 raw response，但不要提交密钥。
5. 外部服务暂时不可用时，应能读取已保存的解析文本或 raw response，重新生成提交的 Auto、schema 和 Cross-check 结果。
6. Final 中的人工修订不要求由代码复现，但必须保留修改前后值、原因和 PDF 证据。

提交前请确认：8 家公司齐全，Gold/Auto/Final 分开，Auto 结果可以重新运行，组内复核已经完成，融资报告中的关键数字能够回到 Final 记录和 PDF 原文。
