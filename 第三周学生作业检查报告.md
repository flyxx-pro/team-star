# 第三周学生作业检查报告

检查时间：2026-06-20  
检查对象：陈雨昂、赵秉清、杨苗鑫、李泽润、尚恬、刘宇轩、苏宇垚七位同学的 Week3 单独提交。  
样本范围：603418 友升股份、301563 云汉芯城、301581 黄山谷捷、688758 赛分科技、688775 影石创新、920100 三协电机、920116 星图测控、001282 三联锻造。

本次检查重点调整为两个问题：

| 检查问题 | 判断标准 |
|---|---|
| 学生是否完成 gold standard 任务 | 是否人工或半人工标出可信 gold；是否有 PDF 页码、原文、真实主体、单位、事件时间；是否覆盖认缴/增资、股权转让、股权结构快照；是否能支持后续外部复核 |
| 学生是否完成自动化任务 | 是否有从 PDF 或页码化文本开始的流程；是否包括解析、定位、候选事件、规则/LLM 抽取、校验、cross-check、失败复核；自动结果是否与人工 gold 分开；是否能一键或分步复现 |

变量名、字段名、目录名不是本次评价的核心。只要能表达同一业务含义，变量名不同不是问题。真正的问题是：gold 是否可信，自动化流程是否能复现，出错后是否能定位和返工。

## 一、总体判断

第三周大家已经意识到任务有两部分：一部分是做可信 gold，一部分是把抽取过程自动化。但从提交结果看，多数同学只完成了其中一部分，或者两部分之间没有闭环。

### 1. Gold standard 方面

目前还没有任何一位同学完整达到“8 家公司全量可信 gold”的标准。相对较接近的是刘宇轩、陈雨昂、杨苗鑫、赵秉清，但各自问题不同：

| 层面 | 当前情况 |
|---|---|
| Gold 是否标出 | 大多数同学都有某种 manual gold 或人工确认结果，但覆盖范围、证据质量和事件边界差异很大 |
| Gold 是否标准 | 刘宇轩覆盖较多，陈雨昂和杨苗鑫有结构化结果，赵秉清有较完整目录；但普遍仍有范围不清、自动/人工混合、事件拆分口径不清的问题 |
| 股权转让是否纳入 | 刘宇轩、杨苗鑫、赵秉清、尚恬、李泽润有一定涉及；陈雨昂和苏宇垚缺口明显 |
| 外部对照复核 | 学生之间 gold 互查、与教师 gold 互查并不是本周原始要求；本报告中的横向对照是本次复核额外做的，用来帮助发现下一步需要回 PDF 核实的口径差异 |

这里最重要的不是“谁的变量名和教师版本一样”，而是他们有没有从 PDF 中确定真实数字。比如三协电机稳正景明、长泽创投的拆分，黄山谷捷股权转让中 78/780 的单位差异，赛分科技股权转让价格和金额差异，这些都必须回 PDF 判断，不能靠字段名解决。

### 2. 自动化流程方面

自动化流程也没有完全达标。较好的方向是赵秉清、杨苗鑫、李泽润，三位同学更明确地写出了 PDF 解析、章节定位、候选抽取、规则/LLM、校验、对比等步骤。刘宇轩有较有价值的 TOC 定位和长文本压缩思路，但最终结果没有把 auto 和 manual 分开。陈雨昂的 Week3 pipeline 比较清楚地说明了校验和 Excel 转换，但它从 Week2 JSONL 开始，不是从 PDF 开始。尚恬有流程骨架，但自动结果质量和复核链条较弱。苏宇垚的正式 zip 提交没有代码；早先 GitHub 目录虽有 `code03_download_pdfsdownload.py.txt` 等代码名文件，但均为空文件，不能视为可复现代码。

自动化的共同短板是：很多流程能“生成文件”，但不能证明“从 PDF 到最终结果可复现”。尤其是校验失败之后，大多数同学没有设计自动返工或人工复核队列的闭环。

## 三、逐人修改建议

### 陈雨昂

提交地址：`https://github.com/Chen-yu516516/cya/pull/3`

Gold 方面：已经有 8 家公司的 Excel gold，但更像“把已有 JSONL 转成表格并做校验”，不是完整的 PDF 人工标注 gold。最大缺口是股权转让，尤其本任务已经明确股权转让也要提取，因为它关系到进入/退出价格。建议补 `share_transfer_flow`，并为每条记录补 PDF 页码、原文、单位和是否计算得出。

最重要修改：把 `manual_gold`、`auto_output`、`comparison` 三层分开。赛分科技单位错误可以作为很好的示范：auto 为什么错，gold 如何修，cross-check 如何发现。

### 3. 公司深讲内容

每位同学讲公司时，至少回答以下问题：

| 问题 | 说明 |
|---|---|
| 这家公司有哪些关键股本变化事件 | 按时间线讲设立、增资、股权转让、整体变更、定向发行等 |
| PDF 原文在哪里 | 至少展示 2-3 处关键 PDF 页码或原文片段，说明数字从哪里来 |

### 5. 评价重点

这次 presentation 不按“文件数量”评价，而看三件事：

| 评价点 | 具体含义 |
|---|---|
| 是否真的理解公司 | 能否讲清楚这家公司上市前股本变化的时间线和关键事件 |
| 是否能从个案提炼方法 | 能否说明这家公司反映了 IPO 招股书抽取中的哪些共性问题 |
| 是否能反思自动化 | 能否讲清楚自己的流程哪里有效、哪里失败、下一步如何让流程更自动、更可信 |

最终希望同学从“机械完成任务”转向“理解问题结构”。一个讲得好的 presentation，应当让听的人明白：如果换一家类似公司，应该先去招股书哪里找信息，哪些数字可以直接抽，哪些数字必须人工判断或 cross-check，自动化流程应该在哪里加规则、加 LLM 或加人工复核。
