# 招股书股本结构抽取 —— 10分钟汇报大纲

> 刘宇轩 & 苏宇垚 | C组（科创板）| 2026.06

---

## Slide 1: 任务是什么（30秒）

**从8家IPO招股书PDF中，把上市前的股本变化变成结构化数据**

- 输入：8份招股书 PDF（主板/创业板/科创板/北交所各2家，400-800页不等）
- 输出：每家公司 3 张表 → 认缴流量 / 股权结构存量 / 股权转让
- 要求：每个数字都能回到 PDF 原文和页码，通过 Schema 校验和数值交叉核对

---

## Slide 2: 我们搭的流水线（1分钟）

```
PDF下载(cninfo爬虫) → MinerU解析 → 章节定位(关键词密度评分) 
→ Deepseek AI抽取 → Pydantic Schema校验 → 数值Cross-check 
→ 人工复核队列 → Excel三表输出
```

**关键技术选择：**
- Deepseek-chat，Temperature=0.1 压制幻觉
- 关键词密度评分代替全文输入（75K → 48K 字符，信息密度提升 10 倍）
- 分轮调用：认缴/存量/转让分别提取，避免 JSON 截断

## Slide 4: 遇到的核心问题（2分钟）—— 重点讲

**问题1：PDF解析失败 → Fallback机制**

三联锻造：MinerU两次解析都失败（200页里"增资"关键词0次）
→ 解决：换页码范围重新上传，第三次成功提取

**问题2：候选文本截断 → 关键词密度评分**

影石创新：全文75K字符，前35K截断丢失了股权结构表
→ 解决：对200页做关键词密度评分，高分页优先保留（影石存量 2→40）

**问题3：JSON输出超长截断 → 分轮调用**

赛分科技：完整Prompt让AI输出超过24K tokens，JSON被截断
→ 解决：认缴流量和股权存量分开两次调用API

**问题4：MinerU表格数据丢失 → null值**

部分公司的认购明细是表格格式，MinerU展平后丢失行列对应关系
→ 当前方案：标记为 manual_review_queue，需人工回PDF补全

## Slide 5: 我们从中学到了什么（1分钟）

1. **"香肠理论"是对的**：每一步只做一件事，验证通过再做下一步。不能一口气把整个PDF扔给AI

2. **AI能抽框架，抽不准数字**：投资者名称、事件结构基本正确，但具体金额/数量在表格中时容易null

3. **校验比抽取更重要**：Schema + Cross-check才能发现错误。光生成JSON不够

4. **PDF解析是瓶颈**：MinerU对扫描版/加密PDF和复杂表格的处理能力决定了整个流程的上限

5. **GitHub提交 ≠ 任务完成**：上周被老师指出"提交规范但结果质量不足"后，才知道要主动做对比和差异分析
