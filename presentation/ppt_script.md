## 第3页：自动化流水线（7步）

- ① PDF下载 — cninfo巨潮爬虫（Session Cookie + UA轮换 + 反爬重试）
- ② MinerU解析 — PDF转结构化JSON（保留页码+坐标）
- ③ TOC目录定位 — 先解析目录页，精准找到"发行人基本情况"起始页
- ④ 章节精准提取 — 只提取目标章节，不全文输入
- ⑤ 关键词密度评分 — 75K→48K字符，信息密度提升10倍
- ⑥ Deepseek AI抽取 — Temperature=0.1，数据大时分轮调用
- ⑦ Pydantic校验 + 数值Cross-check + 人工复核队列

## 第5页：踩坑2 — 表格被MinerU展平，数值全null

- 赛分科技第59页：股权结构表格 → MinerU解析只剩文字碎片
- "认购数量"关键词0次 | "认购金额"0次 | AI完全找不到数字
- 影石创新第64页："股权结构如下："后面表格完全丢失
- 解法：人机协作
  - 可自动 → 关键词密度评分 + TOC定位 + AI提取框架
  - 需人工 → PDF表格拍照OCR/手动填入 → Markdown模板 → 秒转JSONL
- 最终：赛分184条存量、影石65条存量，全部有数值

## 第6页：踩坑3 — AI输出JSON截断 + 数据重复

- 赛分科技存量数据太大，单次API输出超24K tokens → JSON被截断
- 认缴记录同一条被AI重复生成15次
- 解法：
  - 分轮调用：认缴流量和股权存量分开两次API请求
  - 去重：同一认购方+同一日期 → 只保留一条
- 修复后：赛分15认缴+184存量，零null、零重复

## 第7页：核心创新 — 目录定位法（TOC-based）

- 步骤：
  - 1. 扫描MinerU JSON前30页，找到目录页
  - 2. 正则提取："第四节 发行人基本情况 ........... 30"
  - 3. 找下一节起始页作为边界（如"第五节 业务和技术 ........... 87"）
  - 4. 只提取第30-87页文本
- 效果：三协电机存量 1条→37条（原来提取的是错误页面）
- 可复用到任何招股书，不依赖人工经验
