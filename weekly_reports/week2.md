## 4. 代码与工具

| 组件 | 路径 | 功能 |
|------|------|------|
| Pydantic Schema | `schemas/extraction_models.py` | 定义两类记录字段、类型、必填项和校验规则 |
| AI 抽取 | `scripts/extract_week2.py` | 从MinerU JSON提取JSONL（Deepseek-chat, T=0.1） |
| JSONL→Excel | `scripts/jsonl_to_excel.py` | 将JSONL转换为三表格式Excel |
| 校验+Cross-check | `scripts/validate_jsonl.py` | Schema校验 + 总量核对 + 逐股东核对 |
| 巨潮爬虫 | `cninfo_spider.py` | PDF批量下载（含反爬机制） |

运行方式：
```bash
py scripts/extract_week2.py      # Step 1: AI抽取
py scripts/validate_jsonl.py     # Step 2: Schema+交叉校验
py scripts/jsonl_to_excel.py     # Step 3: 生成Excel
```

## 5. Schema校验与Cross-check结果

所有7家公司通过 Pydantic Schema 校验：

- JSON 解析：7/7 通过
- record_type 检查：全部记录正确标记为 subscription_flow 或 equity_snapshot
- 必填字段检查：PDF页码、认购方/股东名称、原文证据全部非空
- 数字类型检查：认购数量、认购金额、持股数、持股比例均为数值或null
- t0 存在性：7/7 均有 t0 股权结构
- 持股比例范围：所有比例在0-100以内

Cross-check（友升股份为例）：
- 2020-09-27 增资：t0 股本12000 + 新增1380 = t1 股本13380 ✓（差额0）
- 2022-12-19 增资：t1 股本13380 + 新增1100.1333 = t2 股本14480.1333 ✓（差额0）
- 逐股东：金浦临港基金 t0持股0 → t1增资840 → t1持股840 ✓
- 老股东稀释：泽升贸易 t0持股8976 → t1仍持股8976（未参与增资，比例从74.8%降至67.08%）
