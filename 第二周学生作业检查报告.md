# 第二周学生作业检查报告

检查时间：2026-06-13  
检查依据：`第一周学生作业总体反馈.md` 中 Week 2 统一提交要求，以及教师基础示例和本轮复核口径。  
检查对象：李泽润、赵秉清、杨苗鑫、刘宇轩苏宇垚、尚恬、陈雨昂六组 Week2 提交。

第二周大家都投入了很多工作，也已经从“下载 PDF、解析文本、生成 JSON”推进到“证据回源、schema 校验、cross-check”的阶段。但从目前提交看，距离这个任务真正完整完成还有明显距离。核心原因不是大家不努力，而是任务目标本身还没有被充分拆开。

这个项目实际有两层任务：

- 第一层是从 PDF 中找出可信的 gold standard。也就是人工或半人工地确认：字段在哪里、页码在哪里、原文怎么写、每个数字应当怎样定义、哪些事件属于增资、哪些属于股权转让、哪些只是整体变更或存量快照。
- 第二层是把人工做 gold standard 的过程尽可能自动化。也就是设计一条可复现流程，让代码或 AI agent 能从 PDF 开始，自动完成解析、章节定位、候选事件识别、结构化抽取、schema 校验、数值 cross-check、失败重试和人工复核队列。

第二周多数提交更偏第一层的一部分：有些组做出了较多结构化结果，有些组做了批注和校验。但第二层还普遍不够，也就是“这些结果到底是不是代码自动做出来的”“规则能覆盖哪些场景”“LLM 提示词是否稳定”“校验失败后如何返工”还没有讲清楚。第三周需要继续围绕同一 8 家公司，把这两层任务一起补完整。

## 一、总体评语

### 1. 大家已经进入了正确问题域，但理解还不够完整

第二周明显比第一周更接近任务本质：大家开始关注上市前股本变化、股权结构存量表、增资认缴流量表、JSONL、Excel 查看版、schema 和 cross-check。这个方向是对的。

但现在仍有一个共同误区：把“交出一批 JSON/Excel”理解为任务完成。实际上，JSON/Excel 只是结果形式。真正要证明的是两个问题：

- 这些数字能不能作为 gold standard，经得起 PDF 回源复核。
- 这些数字能不能通过一套流程自动或半自动复现，而不是只依赖人工整理。

如果第一步定义不清楚，后面自动化跑得再快也只是把错误放大。如果只有人工整理，没有流程自动化，那么样本一扩大也无法支撑研究项目。

### 2. 第二周比较好的进展

多数同学已经补齐了公共样本的 JSONL 输出，部分同学也提交了 Excel、schema、校验脚本和证据页码。李泽润和尚恬提交了批注 PDF，这对复核帮助很大。赵秉清和陈雨昂做了较大规模的流程尝试，说明有自动化意识。刘宇轩、苏宇垚组保留了较清楚的字段和证据思路。杨苗鑫的提交虽然校验链没有闭合，但能保留 fail 项，也比把问题隐藏起来更有价值。

这些都是可以继续迭代的基础。第三周不是推倒重来，而是把现有工作从“结果文件”推进到“可信 gold + 可复现流程”。

### 3. 第二周最主要的问题

最主要的问题是 gold standard 与自动化流程混在一起了。

有些结果看起来较完整，但不清楚哪些是人工确认的 gold，哪些是机器自动抽取的结果，哪些又是人工修补后的结果。有些仓库有脚本，但脚本不能从原始 PDF 自动复现最终 JSON。有些结果有 cross-check，但只有 pass/fail，没有保留具体数字。有些 LLM 或 agent 参与了抽取，但没有提交 prompt、输入文本、模型参数和原始输出。

第三周需要把这几类东西明确分开：

- `manual_gold`：人工基准答案，允许人工批注、人工查 PDF、人工修正，但必须保留证据。
- `auto_output`：代码或 LLM 自动跑出的结果，不能和人工修正混在一起。
- `comparison`：自动结果与人工 gold 的差异表，说明哪些自动抽对了、哪些漏了、哪些错了、哪些需要人工复核。
- `pipeline`：从 PDF 到自动结果的可运行代码、规则、prompt 和日志。

## 二、共同修改意见

### 1. 先把人工 gold standard 做成可审计结果

第三周每组都要继续围绕同一 8 家公司，建立自己的人工 gold standard。这里的“人工”不是随便手填，而是要像做数据集标注一样，保留页码、原文、字段定义和修改痕迹。

最低要求是：

- 每条记录必须能回到 PDF 页码和原文证据。
- 每个字段要说明单位，例如万股、万元、元/股、百分比。
- PDF 未直接披露、需要计算的字段不能伪装成原文披露，必须标注 `calculated` 或放入校验表。
- 不确定的记录不要硬填为正确，应放入 `manual_review_queue.csv`。
- 可以参考教师提供的基础示例和基准口径，但不能无脑复制；如果发现示例或基准可能有问题，可以提出修正，前提是给出 PDF 页码、原文证据和理由。

### 2. 再把人工流程拆成可自动化步骤

第三周每组都要明确写出并提交一条自动化流水线。可以是规则为主，也可以是 LLM 为主，也可以是规则 + LLM + 人工复核队列，但必须能说明每一步做什么。

建议流程如下：

- PDF 下载或读取。
- PDF 转 Markdown/Text，并保留页码。
- 目录和章节识别。
- 目标章节定位，例如历史沿革、股本形成及变化、历次增资、历次股权转让、新增股东。
- 候选事件包生成，按小标题或表格边界切块，而不是全页乱截。
- 结构化抽取，输出 `subscription_flow`、`share_transfer_flow`、`equity_snapshot`。
- Pydantic schema 校验。
- 数值 cross-check。
- 自动结果与人工 gold 对比。
- 失败样本进入人工复核队列。

如果某一步目前必须人工完成，也可以接受，但要把人工介入点写清楚：人工改了什么、为什么改、规则为什么失败、下次是否可以自动化。

### 3. 规则提取可以接受，但要说明覆盖范围和失败边界

规则提取不是低级方法。对招股说明书来说，很多高质量流程都需要规则定位章节、识别标题、提取表格、做单位转换和数值校验。

但规则方法必须回答：

- 规则覆盖了哪些章节标题。
- 规则能处理哪些公司和哪些事件类型。
- 哪些事件靠规则可以稳定抽取。
- 哪些事件规则抽不到，需要 LLM 或人工复核。
- 规则误抽了哪些内容，如何过滤。
- 规则在 8 家公司上的覆盖率、漏抽率和误抽率是多少。

第三周不要求规则一次覆盖所有复杂情况，但要求把“规则能做什么、不能做什么”讲清楚。

### 4. 使用 LLM 或 agent 时，必须提交 prompt 和敏感性分析

如果使用 LLM、agent、file-agent、ChatGPT、Qwen、DeepSeek 或其他模型，不能只提交最终 JSON。必须提交：

- 模型名称和版本。
- system prompt。
- user prompt 模板。
- temperature、max tokens 等参数。
- 输入候选文本或 chunk 文件。
- 原始模型输出。
- 解析失败或 schema 失败日志。
- 至少一次 prompt 修改前后的效果对比。

第三周希望大家思考更高自动化程度的 agent 流程。例如模型不仅抽取 JSON，还能在 schema 失败后自动回到原文重试，在 cross-check 不通过时自动定位相关表格，在不确定时把问题写入人工复核队列。这种“能自主发现问题并请求复核”的设计，才更接近外部 agent 比赛中对自动化能力的要求。

### 5. Cross-check 必须出现数字

第三周的校验不能只写 `pass/fail`。必须保留数字，否则无法判断为什么通过或失败。

最低数字列包括：

- 上一时点总股本或注册资本。
- 本次新增或转让数量。
- 本次新增或转让金额。
- 本次价格。
- 预期下一时点总股本或注册资本。
- PDF 披露的下一时点总股本或注册资本。
- 差额。
- 逐股东核对结果。
- 校验状态。

如果是股权转让，不能用“增资后总股本增加”去校验，因为股权转让不改变总股本。应校验转让方减少、受让方增加、总量不变。

### 6. 自动化不是要求完全无人，但要留下人工介入证据

这个项目可以接受 human-in-the-loop。关键是不能让人工介入变成黑箱。

如果人工修正了自动结果，需要提交：

- 自动抽取原始结果。
- 人工修正后结果。
- 修正 diff。
- 修正原因。
- PDF 页码和原文证据。
- 这个错误未来是否可以通过规则、prompt 或 schema 避免。

## 五、第三周验收标准

第三周验收不只看最终 JSON 是否好看，而看完整闭环。

最低通过标准：

- 8 家公司全部覆盖。
- 人工 gold、自动输出、比较结果三者分开。
- 每条人工 gold 记录有 PDF 页码和原文证据。
- 自动流程能运行，且不依赖本机绝对路径。
- 有 Pydantic schema 校验。
- 有带数字的 cross-check。
- 有自动结果与人工 gold 的差异表。
- 有 prompt 或规则说明。
- 有失败样本和人工复核队列。

较好标准：

- 自动流程能稳定定位章节和候选事件包。
- 能区分增资认缴、股权转让、整体变更、设立出资、资本公积转增等事件类型。
- 能在 schema 或 cross-check 失败后自动重试或自动生成复核任务。
- 能清楚说明规则覆盖率和 LLM 提示词敏感性。
- 能发现并解释与教师基础示例或基准口径不一致的项目。

优秀标准：

- 接近 agent 化流程：模型或程序能自主检查缺口、回 PDF 找证据、修正单位、重新抽取、生成复核队列。
- 自动结果与人工 gold 的差异可量化，可解释，可迭代。
- 流程有扩展到更多招股说明书的设计，而不是只为 8 家手工定制。

第三周的目标不是把所有问题一次性消灭，而是让每组真正理解：这个项目最终要产出的不是一张表，而是一套可信、可复核、可扩展的 IPO 招股说明书股本变化抽取方法。
