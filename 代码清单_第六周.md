# 第六周代码清单

> 提交说明中可以直接复制本清单。所有路径均为仓库相对路径。

## 1. 数据提取主流程（核心）

```text
pipeline/run_week6_pipeline.py
```

作用：统一入口调用的核心抽取脚本，从 `data/manifest/company_manifest.csv` 和 `data/raw_response/*.jsonl` 读取页码化 PDF/Markdown 证据，生成：

- `auto_output/subscription_auto.csv`
- `auto_output/transfer_auto.csv`
- `auto_output/equity_snapshot_auto.csv`
- `validation/schema_result.csv`
- `validation/cross_check.csv`
- `validation/review_queue.csv`
- `logs/run_log.csv`
- `logs/file_hashes.csv`

关键函数：

- `load_raw_records()`：读取 raw response，并按 manifest 固定 8 家公司。
- `write_raw_markdown()`：把页码化 raw response 转成可审计 Markdown 证据。
- `build_subscription_auto()`：生成认缴/增资 Auto 表。
- `build_snapshot_auto()`：生成股权结构快照 Auto 表。
- `build_transfer_auto()`：生成股权转让候选 Auto 表。
- `build_cross_check()`：生成自动校验结果。

统一运行命令：

```bash
python pipeline/run_all.py --root .
```

## 2. Markdown 表格解析（含 rowspan/colspan）

```text
pipeline/markdown_table_parser.py
```

作用：解析 MinerU/Markdown 中的表格。如果是普通 Markdown pipe table，则按 `|` 切分；如果是 HTML table，则读取 `<td>/<th>` 的 `rowspan` 和 `colspan`，并展开为规则矩形表，便于后续按列抽取。

关键函数：

- `parse_tables_from_markdown()`：统一返回所有 Markdown/HTML 表格。
- `parse_html_tables()`：解析 HTML 表格。
- `expand_spans()`：展开 `rowspan/colspan`，把跨行跨列表格补成矩形。
- `parse_pipe_tables()`：解析普通 Markdown 表格。

示例命令：

```bash
python pipeline/markdown_table_parser.py data/raw_text/301563_云汉芯城_parsed_evidence.md --out-dir auto_output/parsed_markdown_tables --json auto_output/parsed_markdown_tables.json
```

## 3. 三表分类逻辑（认缴/快照/转让）

```text
pipeline/run_week6_pipeline.py
```

对应代码位置：

- `build_subscription_auto()`：识别 `subscription_flow` 记录，生成“认缴/增资表”。
- `build_snapshot_auto()`：识别 `equity_snapshot` 记录，生成“股权结构存量/快照表”。
- `build_transfer_auto()`：基于证据文本中的“转让、受让、股权转让、转让协议、代持解除”等关键词生成“股权转让候选表”。

分类原则：

- 认缴/增资：有投资方、认缴金额、认购数量、增资日期、入股价格等字段，进入 `subscription_auto.csv`。
- 股权快照：有时点、股东名称、持股数、出资额、持股比例等字段，进入 `equity_snapshot_auto.csv`。
- 股权转让：必须先作为候选进入 `transfer_auto.csv`；Final 只保留能确认转让方、受让方和关键数字的记录，误命中留在 `review/transfer_false_positive_review.csv`。
- PDF 未披露字段留空，不用 0 补齐。

## 4. Cross-check 脚本

```text
pipeline/cross_check.py
```

作用：单独重跑交叉校验，不必重新生成全部 Auto 表。

运行命令：

```bash
python pipeline/cross_check.py --root .
```

实际复用的核心函数位于：

```text
pipeline/run_week6_pipeline.py
```

关键函数：

- `build_cross_check()`：检查认缴金额/认购数量/单价是否一致；检查同一股权快照时点的持股比例合计是否接近 100%。

输出文件：

```text
validation/cross_check.csv
```

pipeline/curate_final_and_report.py
